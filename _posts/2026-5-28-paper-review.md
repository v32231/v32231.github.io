# 논문 선정 이유

AI 에이전트가 발전함에 따라 AI 에이전트 스스로 수행할 수 있는 작업의 범위가 점점 더 넓어지고 있다. 이전에는 사용자의 질문에 답을 하는 정도에 그쳤지만 이제는 컴퓨터를 스스로 조작해 파일을 생성, 수정, 삭제하거나 프로그램을 다운 받는 등 정말 많은 작업을 수행할 수 있게 되었다. 하지만 AI 에이전트는 항상 사용자가 원하는 작업만을 수행하지 않으며, AI 에이전트가 할 수 있는 작업이 늘어남에 따라 그로 인해 발생할 수 있는 위험도 훨씬 많아졌다. 적어도 AI로 발생하는 ‘치명적인’ 위험을 피하기 위해서는 적절한 권한 설정이 중요하다고 생각하고 있다. 하지만 나도 Codex와 같은 AI 에이전트를 자주 사용하면서도 적절한 권한 설정을 하고 있지 않다고 생각한다. 고민하면서도 언제나 찝찝함을 느끼면서도 full access를 허용하게 되는 자신을 돌아보며 좀더 쉽게 권한 설정을 할 수 있는 방법에 대한 논문이라고 생각해서 선정하게 되었다.

# Introduction

- 연구 배경
    - AI 에이전트만을 위한 권한 관리 시스템이 필요하다.
- 기존 방식의 한계
    - 기존 시스템의 설치 시점 권한 부여 방식은 실행 계획이 동적으로 만들어진다는 AI 에이전트 특성 상 지나치게 넓은 권한을 허용하거나 실제 상황에 맞지 않는 결정을 유도할 수 있다.
    - 기존 시스템의 런타임 권한 부여 방식은 하나의 작업을 수행하는 과정에서 여러 도구와 데이터에 연속적으로 접근할 수도 있는 AI 에이전트 특성 상 사용자의 작업 흐름이 반복적으로 중단될 수 있다.
    - 기존 모델은 이전에 보지 못한 데이터 유형이나 작업 과정에 유연하게 대응하기 어렵다.
    - 외부 도구가 필요 이상의 많은 데이터를 요구하거나 AI 에이전트가 자연어를 잘못 해석하는 등 동적인 위험을 세밀하게 통제하기 어렵다.
- 제안하는 해결법
    - AI 에이전트의 데이터 접근 권한을 자동으로 관리하는 권한 보조자를 개발한다.

# Methodology1: 연구 설계

- 저자들은 AI 에이전트 환경에서 사용자가 어떤 기준으로 데이터 접근 권한을 판단하는지 확인하기 위해 비네트 기반 사용자 연구를 진행했다.
    - 비네트 기반 연구는 실제 시스템을 배포하지 않더라도, 구체적인 시나리오를 제시해 참여자의 현실적인 판단을 관찰할 수 있다는 장점이 있다.
- 시나리오
    - 미래형 AI 개인 비서를 훈련시키는 사용자로 가정한다.
        - 해당 시나리오는 선행 연구 흐름을 LLM 기반 AI 에이전트 환경으로 확장한 것이다.
- 참여자는 세 가지 유형의 선택을 한다.
    - 사용자 요청을 해결하기 위해 어떤 도구가 필요한지 선택한다.
    - 요청 해결에 필요한 데이터를 선택한다.
    - 해당 데이터에 AI 에이전트나 도구가 접근해도 되는지 권한을 결정한다.
- 권한 결정 선택지
    - Yes, always share (항상 공유)
    - Yes, but ask me next time (일회성 공유)
    - No, but ask me next time (일회성 거부)
    - No never share (항상 거부)
    - 이 중 '항상 공유'와 '항상 거부'는 AI 에이전트가 이후 비슷한 상황에서 AI 에이전트가 참고할 중요한 기준이 된다.
    
- 도메인
    - 여러 도메인을 사용하여 같은 데이터여도 어떤 상황에서 쓰느냐에 따라 사용자의 결정이 달라질 수 있음을 확인하기 위해서이다.

- AI 에이전트가 항상 올바르게 행동한다고 가정하지 않는다.
    - 권한 결정 질문의 25%에 불필요한 데이터 유형을 포함시켰다.
    - 이는 AI 에이전트나 AI 에이전트가 사용하는 도구가 필요 이상의 데이터를 요구할 수도 있다는 현실적인 위험을 반영한 조건이다.
    - 또한 이를 통해 사용자가 과도한 데이터 요청을 인식할 수 있는 지 확인할 수 있다.

# Methodology2: 권한 선호 분석

- 수집된 응답을 분석하여 사용자의 권한 결정에 영향을 미치는 요인을 분석했다.
    - 이 단계의 목적은 단순히 허용 또는 거부 비율을 계산하는 것이 아니라, 사용자의 권한 선호가 예측 가능한 패턴을 가지는지 확인하는 것이다.
    - 만약 사용자의 권한 결정이 완전히 무작위적이라면, 권한 예측 모델을 설계하기 어렵다.
    - 반대로 반복되는 패턴이 있다면, 사용자의 과거 결정이나 유사 사용자들의 선택을 활용해 향후 권한 결정을 예측할 수 있다.

- 분석 기준
    
    | **분석 기준** | **확인하려는 내용** |
    | --- | --- |
    | 에이전트 실수 여부 | 불필요한 데이터가 포함될 때 사용자가 더 조심스럽게 판단하는지 확인 |
    | 과잉/과소 권한 부여 | 사용자가 필요한 데이터만 적절히 공유하는지 확인 |
    | 도메인/맥락 | 금융, 여행, 엔터테인먼트 등 사용 맥락에 따라 선호가 달라지는지 확인 |
    | 데이터 민감도 | SSN, 여권, 계좌 정보 같은 민감 데이터에 더 보수적으로 반응하는지 확인 |
    | 사용자 특성 | 나이, AI 신뢰도, 프라이버시 의식 등이 권한 결정과 관련 있는지 확인 |
    | 선호 일관성 | 같은 사용자 또는 비슷한 사용자들 사이에 반복되는 권한 패턴이 있는지 확인 |

- 이 분석은 이후 권한 예측 모델 설계의 근거가 된다.
    - 사용자의 권한 선호가 도메인, 데이터 민감도, 에이전트의 실수 여부에 따라 달라지는지 확인한다.
    - 동시에 같은 사용자 안에서 일관된 판단이 나타나는지, 서로 다른 사용자 사이에도 유사한 판단 패턴이 있는지 확인한다.
    - 이러한 일관성과 유사성이 확인되면, 개인 사용자 이력과 유사 사용자 패턴을 활용한 권한 예측 모델을 설계할 수 있다.

# Methodology3: 권한 예측 모델 설계

- 사용자 권한 선호 패턴을 바탕으로, 사용자의 향후 권한 결정을 예측하는 모델을 설계한다.
    - 이 모델의 목적은 사용자가 매번 권한 요청에 직접 응답하지 않아도, AI 에이전트가 사용자의 선호에 가까운 결정을 내리도록 돕는 것이다.
    - 이를 위해 개인 사용자의 권한 이력과 유사 사용자들의 권한 결정 패턴을 함께 활용한다.

- 비교한 모델은 세 가지이다.

    | **모델** | **핵심 아이디어** | **의미** |
    | --- | --- | --- |
    | IC 모델 | LLM 인컨텍스트 학습으로 개인 사용자 이력을 기반으로 예측 | 개인 사용자의 구체적인 선호를 반영 |
    | CF 모델 | 협업 필터링으로 유사 사용자 패턴을 활용 | 개인 이력이 부족할 때 다른 사용자들의 패턴을 활용 |
    | Hybrid 모델 | IC와 CF를 결합 | 개인 선호와 유사 사용자 패턴을 함께 반영 |


- IC 모델
    - LLM에게 사용자의 정보를 자연어 프롬프트로 제공하고, 새로운 데이터 공유 요청에 대해 사용자가 허용할지 거부할지 예측하게 한다.
    - 입력 정보

        | **입력 정보** | **예시** |
        | --- | --- |
        | 인구통계 | 나이, 성별, 교육 수준 |
        | AI 사용 경험 | AI 친숙도, 사용 빈도, AI 신뢰도 |
        | 프라이버시 태도 | 프라이버시 의식 수준, 우려 도메인 |
        | 권한 이력 | 과거 질의, 도구, 데이터 유형, 사용자의 결정 |

    - IC 모델은 LLM이 자연어 맥락을 처리할 수 있기 때문에, 이전에 보지 못한 데이터 유형이나 새로운 상황에도 비교적 유연하게 대응할 수 있다.
    - 다만 개인 사용자의 권한 이력이 부족하면 그 사용자의 실제 선호를 충분히 파악하기 어렵다.

- CF 모델
    - CF 모델은 협업 필터링을 사용하여 비슷한 사용자들이 과거에 어떤 권한 결정을 했는지를 바탕으로 현재 사용자의 선택을 예측한다.
    - 논문에서는 LightGCN이라는 그래프 기반 협업 필터링 모델을 사용한다.
    - 이 방식은 개인 사용자의 권한 이력이 많지 않은 초기 상황에서도 유사 사용자들의 패턴을 활용할 수 있다는 장점이 있다.
    - 그러나 이전에 본 적 없는 새로운 데이터 유형이나 새로운 맥락에는 대응하기 어렵다.

- Hybrid 모델
    - Hybrid 모델은 IC와 CF를 결합한 방식이다.
    - 먼저 CF 모델이 유사 사용자 기반으로 권한 추천을 만들고, 이 중 신뢰도가 높은 추천만 LLM 프롬프트에 추가한다.
    - 이후 LLM은 개인 사용자 정보, 권한 이력, CF 추천을 함께 참고해 최종 권한 결정을 예측한다.
    - 이 방식은 개인 선호를 반영하는 IC의 장점과 유사 사용자 패턴을 활용하는 CF의 장점을 함께 사용한다.
    - 다만 CF 추천이 부정확하면 최종 예측에도 영향을 줄 수 있기 때문에, 논문에서는 신뢰도가 높은 CF 추천만 사용했다.

# Results

# Results

- 사용자 연구 결과, 참여자들은 AI 에이전트가 불필요한 데이터를 요청하는 상황에서 더 보수적인 태도를 보였다.

    ![pr](/image/2026-05-28/0528-pr3.png)

    필요한 데이터만 제시되었을 때 “항상 공유”를 한 번도 선택하지 않은 참여자는 17.2%였지만, 불필요한 데이터가 함께 제시되었을 때는 25.6%로 증가했다. 반대로 “절대 공유하지 않기”를 한 번도 선택하지 않은 참여자는 32.0%에서 21.2%로 감소했다. 이는 사용자가 AI 에이전트의 판단을 무조건 신뢰하지 않으며, 에이전트가 필요 이상의 데이터를 요구한다고 느끼면 자동 공유를 더 꺼린다는 것을 보여준다.

- 또한 과잉 권한 부여가 과소 권한 부여보다 더 흔하게 나타났다.

    ![pr](/image/2026-05-28/0528-pr4.png)

    참여자의 90%는 자신이 받은 데이터 공유 요청 중 15.4% 이상에서 과잉 권한을 부여했다. 반면 모든 데이터 유형에 대해 완전히 적절한 권한을 부여한 참여자는 없었고, 90% 이상의 데이터 유형에 대해 적절한 권한을 부여한 사용자도 18.7%에 그쳤다. 이는 사용자가 직접 권한을 선택하더라도 필요한 데이터와 불필요한 데이터를 항상 정확히 구분하지 못하며, AI 에이전트 환경에서도 기존 권한 시스템의 과잉 권한 문제가 반복될 수 있음을 의미한다.

- 도메인에 따른 권한 선호 차이도 뚜렷했다.

    ![pr](/image/2026-05-28/0528-pr1.png)

    엔터테인먼트 도메인에서는 “항상 공유” 비율이 55.6%로 가장 높았지만, 금융 도메인에서는 22.2%로 가장 낮았다. 반대로 “절대 공유하지 않기”는 엔터테인먼트에서 2.9%, 금융에서 16.9%로 나타났다. 이는 사용자의 권한 결정이 데이터 자체뿐 아니라 데이터가 사용되는 맥락과 민감도에 따라 달라진다는 것을 보여준다.

- 사용자의 권한 선호는 완전히 무작위적이지 않았다. 엔터테인먼트와 스마트홈 도메인에서는 84.5~85.6%의 참여자가 해당 도메인 안에서 일관된 권한 선호를 보였다. 또한 전체 참여자의 46.3%는 도메인별 권한 허용률의 표준편차가 0.1 미만이었고, 73.4%는 0.2 미만이었다. 이는 많은 사용자가 도메인이 달라져도 비교적 안정적인 권한 판단 패턴을 가진다는 것을 의미한다.

- 사용자 간에도 유사한 권한 패턴이 관찰되었다.

    ![pr](/image/2026-05-28/0528-pr5.png)

    같은 권한 요청에 답한 참여자 쌍의 69.7%는 데이터 유형의 60% 이상에서 유사한 데이터 공유 선호를 보였다. 이는 개인 사용자의 권한 이력이 부족하더라도, 비슷한 사용자들의 결정 패턴을 활용해 권한 결정을 예측할 수 있다는 근거가 된다.

- 모델 평가에서는 Hybrid 모델이 가장 좋은 성능을 보였다.

    | 모델 | 정확도 | 정밀도 | 재현율 | F1 점수 |
    | --- | --- | --- | --- | --- |
    | CF | 83.3% | 91.9% | 83.3% | 87.4% |
    | IC | 84.4% | 91.5% | 85.4% | 88.3% |
    | Hybrid | **85.1%** | **92.8%** | 85.2% | **88.8%** |

    Hybrid 모델은 정확도 85.1%, 정밀도 92.8%, F1 점수 88.8%를 기록했다. 이는 개인 사용자 이력만 활용하는 IC 모델이나 유사 사용자 패턴만 활용하는 CF 모델보다, 두 정보를 함께 사용하는 방식이 더 효과적이라는 것을 의미한다. 특히 정밀도가 가장 높다는 점이 중요하다. 권한 관리에서는 모델이 잘못해서 민감 데이터를 공유하도록 허용하는 것이 큰 위험이기 때문에, “공유해도 된다”고 예측한 경우 실제 사용자도 공유를 허용할 가능성이 높다는 것은 실용적으로 의미가 있다.


- 고신뢰 예측은 정확도를 높이지만 자동화 범위를 줄인다.
    ![pr](/image/2026-05-28/0528-pr6.png)

    신뢰도 임계값을 높여 모델이 확신하는 경우에만 자동 예측하도록 하면, Hybrid 모델의 정확도는 94.4%까지 증가했다. 다만 이 경우 자동으로 처리할 수 있는 요청은 전체의 25.9%로 줄어들었다. 이는 정확도와 자동화 범위 사이에 trade-off가 있음을 보여준다. 따라서 실제 시스템에서는 모든 권한 결정을 자동화하기보다, 확신도가 높은 요청은 자동 처리하고 불확실하거나 민감한 요청은 사용자에게 확인하는 방식이 적절하다.

- 또한 권한 이력이 전혀 없을 때 모델 정확도는 66.9%였지만, 1~4개의 권한 이력만 추가해도 정확도는 77.7%로 증가했다. 이는 권한 보조자가 많은 학습 데이터를 요구하지 않더라도, 사용자의 초기 몇 번의 선택만으로 빠르게 개인화될 수 있음을 보여준다.

- 정리
    - AI 에이전트의 권한 결정이 완전히 자동화되기는 어렵지만, 일정 부분 자동 예측이 가능하다는 것을 보여준다. 사용자의 권한 선호는 도메인과 데이터 민감도에 따라 달라지지만, 동시에 개인 내부의 일관성과 사용자 간 유사성을 가진다. 따라서 개인 권한 이력과 유사 사용자 패턴을 함께 활용하면, AI 에이전트 환경에서 반복적인 권한 요청을 줄이면서도 사용자의 프라이버시 선호를 어느 정도 반영할 수 있다.

# Limitations

- 실제 사용 환경이 아니라 비네트 기반 실험이다. 참여자가 설문 상황에서 고른 선택이 실제 사용 상황의 행동과 완전히 같다고 보기는 어렵다.
- 권한 예측은 했지만, 실제 시스템에서 안전하게 집행하는 문제는 완벽히 해결되지 않았다.
- 사용자의 선호가 도메인마다 크게 달라지는 경우 예측이 어렵다.

# 느낀점

AI 에이전트의 권한 문제를 AI 에이전트를 통해 해결하겠다는 아이디어가 신선했다. 미래를 완벽히 유추해낼 수는 없고, AI 에이전트를 완벽히 신뢰할 수도 없기 때문에 한계가 정말 명확한 연구라고 생각한다. 다만, 자동화 AI 에이전트의 특성 상 기존의 권한 부여 시스템이 부족하다는 주장 자체는 설득력이 있다고 생각한다. 기존 권한 부여 시스템은 메뉴얼화된 시스템 안에서 정해진 권한을 사용자에게 승인 받는 식이었는데 AI 에이전트는 그 상황 자체가 좀더 다양할 수 있다는 부분에서 AI 에이전트를 위한 더 나은 권한 부여 시스템은 뭐가 있을지 생각해보게 되었다. 개인적으로 미리 사용자에게 항상 사용해도 괜찮은 권한을 받고, 그 외의 권한은 사용자에게 계속해서 물어보는 게 아직까지는 보안적인 측면에서 최선이라고 생각한다. 편의성을 고려하면 그게 최선은 아니겠지만 말이다.

논문에서 설명한 AI 에이전트를 사용한 보조 프로그램은 완벽하지는 않지만 비전공자나 AI에 익숙하지 않은 사용자를 대상으로는 쓸모가 있을 거 같다는 생각이 들었다. 완벽히는 아니지만 AI로 인한 위협에 노출될 가능성을 어느정도 줄이는 것까지는 기대해 볼 수 있을 거라고 생각한다.

전부터 AI 에이전트를 사용하면서 권한 부여 시스템에 대해 관심이 있었기 때문에 관련된 좀더 독특한 아이디어는 없는 지 찾아보고 싶다. 찾아보니까 AI 권한 관련 논문이 꽤나 많다는 것을 알았다. ‘Taming Various Privilege Escalation in LLM-Based Agent Systems: A Mandatory Access Control Framework’ 라는 논문은 AI가 과도한 권한을 요구하는 문제를 다루는 것 같은데 이것도 꽤 흥미가 있다. 그 외에도 권한 관련 문제니 보안 정책 쪽도 좀더 찾아봐도 재밌을 거 같다.
